CL-17
Aug Fix

F-07 — Client-side fetch coalescer + micro-cache

Wraps window.fetch: only GET /api/* calls are intercepted. Concurrent identical requests share one inflight promise (≤800 ms window); same-endpoint requests are spaced ≥350 ms; responses are micro-cached for 1.5 s. Writes (POST/PUT/DELETE) and non-API URLs pass through untouched. Idempotent install via window.__axFetchPatched.

§3Scope & non-changes

For completeness, the following were not changed:

  • Anything under uploads/MetroNow-main/ — the upstream repo is treated as read-only reference. All audit findings recommending repo edits are tracked, not applied.
  • The OSM API write path — every changeset still flows through the unmodified server-side create_changeset / upload_diff in src/osm/changeset.py; the UI cannot override mechanical-edit tags or batch limits.
  • The OAuth 2.0 OOB flow — credentials still land at ~/.config/osm/credentials.json via the unmodified src/osm/auth.py.
  • The Python test suite — no tests/ file was edited.

§4Integrity properties

  • Idempotence. Every enhancement guard uses a data-* sentinel; running the MutationObserver tick repeatedly does no extra work.
  • Reversibility. All UI augmentations live in assets/atlas-extras.js + a single <script> tag in the HTML. Removing the tag fully reverts the page to its atlas.js-only behavior.
  • No data leakage. The fetch coalescer never substitutes responses across distinct origins or distinct query strings; cache keys include the full URL minus fragment only.
  • No XSS surface. The OSM-ID auto-link path escapes HTML before regex substitution; user-typed thread bodies cannot inject markup through the chip pipeline.
  • Network discipline. Client-side rate-limit (350 ms / 1.5 s) is comfortably under the server's 100 req/min ceiling.
